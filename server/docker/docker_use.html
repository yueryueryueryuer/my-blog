# Docker 常用命令

  1. 查看正在运行的容器
sudo docker ps
  1. 查看所有的容器
sudo docker ps -a
  1. 查看本地镜像
sudo docker images
  1. 从镜像中运行/停止一个新实例
sudo docker run/stop --help

sudo docker run/stop container
  1. 避免输出 Sudo

这里把当前用户加入到 docker 组就可以直接使用命令，而不用每次都加 sudo

sudo groupadd docker

改完后需要重新登陆用户

sudo gpasswd -a ${USER} docker
  1. Docker 版本
sudo docker --version
  1. 搜索 Docker Image
docker search tutorial

docker Hub

  1. 通过 docker 命令下载 tutorial 镜像
docker pull learn/tutorial
  1. 从指定 image 里生成一个 container 并在其中运行一个命令
docker run [image] [cmd]
  1. 在 container 里运行交互式命令，比如 shell
docker run -i -t [image] [cmd]

docker run -i -t ubuntu /bin/bash
  1. 在 container 里运行后台任务
docker run -d [image] [cmd]
  1. 列出最近一个运行过的 container

不加-l 则只列出正在运行的 container（比如后台任务）

docker ps -l
  1. 列出所有 container
docker ps -a
  1. 查看 container 详情
docker inspect [container]
  1. 删除某个 container

其中 container_id 不需要输入完整，只要能保证唯一即可。

运行中的 Docker 容器是无法删除的，必须先通过 docker stop 或者 docker kill 命令停止。

docker rm [container]

docker rm `docker ps -a -q` 删除所有容器，-q表示只返回容器的ID
  1. 再次运行某个 container
docker start [container]
  1. 查看某个 container 的运行日志
docker logs [container]

docker logs -f [container] 类似tailf
  1. 切换到后台任务 container, 需要当前容器正常运行

注意：切换到后台任务以后无法用Ctrl+C 退出

docker attach [container]
  1. 中止后台任务 container
docker stop [container]
  1. 将 container 保存为一个 image
docker commit [container] [image_name]
  1. 将 image 上传到仓库
docker push [image_name]
  1. 删除 images
docker rmi [image id]
  1. 为容器指定名称，容器的名称是唯一
docker run --name edison -i -t ubuntu /bin/bash
  1. 有三种方式可以唯一指代容器

短 UUID: 716d3c16dc65（12 位）

长 UUID：716d3c16dc654230ada14f555faadd036474231dfca0ca44b597574a5c618565（64 位）

名称: edison

  1. 当前 Docker 宿主机的信息
docker info
  1. 查看容器内部的进程信息
docker top [container]
  1. 在容器中运行后台任务，只对正在运行的容器有效。
docker exec -d [container] [cmd]

docker exec -d edison touch /home/haha
  1. 在容器中运行交付式任务，只对正在运行的容器有效。
docker exec -t -i edison /bin/bash

注：在/var/lib/docker中，可以查看 Docker Image、Container 和 Volumes 等细节信息。

  1. 进入容器
docker exec -it [container] /bin/bash

docker cp dingtalk-work-message:/dingTalkWorkMessageLog/dingTalkWorkMessage.2021-01-11.log /home/zgyxapp/
